Logic Description

Every single object in the game with have its own class. Objects such as the player ship, the asteroids, and bullet projectiles with inherit my GameObj class which will store the objects 2d position, rotation, collision radius, and drawn sprite, and methods related to these attributes. The player ship will listen for user inputs and apply a impulse which is added to a linear, and a radial velocity variable. The ship's sprite will also be updated based on user action. The ship check if it's radius intersects with each asteroid's radius, and if it intersects, it will get destroyed and signal game over. If the ship exceeds the world boundary, it will wrap around the other side. The asteroids will spawn out of bounds and be given an initial velocity towards the center with an offset. The asteroids will despawn once far enough out of bounds. The asteroids will spawn in multiple smaller asteroids when destroyed. The asteroids will be destoyed completely if small enough. Bullets are spawned at the front of the ship when user fires. The bullet is given a constant initial velocity with the direction of the ship. If the bullet's radius intersects with an asteroid's, the asteroid with take damage with an applied impulse and origin of impulse. There will be a GameWorld class that will be referenced by the player ship, asteroids, and bullets. The asteroids will be referenced in the GameWorld object. Functions used to check collision radius intersections and sprite drawing will be handled in the GameWorld object. GameWorld will handle score keeping. GameWorld will have a simple game state engine to handle when the game has not stated, is currently running, or has ended.
